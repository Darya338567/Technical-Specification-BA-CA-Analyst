# Задание 2: Публикация товара на маркетплейсе

---

## 1. User Story (верхнеуровневая постановка)

&gt; **Как** продавец на маркетплейсе,
&gt; **я хочу** опубликовать свой товар на витрине,
&gt; **чтобы** покупатели могли его видеть, изучать характеристики и совершать покупку.

---

## 2. Критерии приёмки

| № | Критерий |
|:---|:---|
| 1 | Продавец может создать карточку товара с названием, описанием, фото, ценой и остатком |
| 2 | Товар отображается на витрине только после модерации |
| 3 | Продавец видит статус публикации: черновик / на модерации / опубликован / отклонён |
| 4 | При отклонении продавец получает причину, может исправить ошибки и отправить повторно |

---

## 3. Use Cases (варианты использования)

| ID | Название | Автор | Описание |
|:---|:---|:---|:---|
| UC-1 | Создать карточку товара | Продавец | Продавец заполняет форму: название, описание, категория, фото, цена, количество |
| UC-2 | Редактировать карточку товара | Продавец | Продавец изменяет данные товара в статусе «Черновик» или после отклонения |
| UC-3 | Отправить на модерацию | Продавец | Продавец отправляет заполненную карточку на проверку |
| UC-4 | Провести модерацию | Система / Модератор | Проверка соответствия правилам площадки (запрещённые товары, корректность описания, качество фото) |
| UC-5 | Опубликовать товар | Система | Товар появляется на витрине, виден покупателям |
| UC-6 | Отклонить товар | Модератор | Указание причины отклонения, возврат продавцу на доработку |
| UC-7 | Снять товар с публикации | Продавец | Товар скрывается с витрины, но остаётся в личном кабинете |
| UC-8 | Просмотреть статус публикации | Продавец | Продавец видит, на каком этапе находится его товар |

---

## 4. Детализация ключевых Use Cases

### UC-1: Создать карточку товара

**Предусловие:** Продавец авторизован в Личном кабинете.

**Основной сценарий:**

1. Продавец нажимает кнопку «Добавить товар»
2. Система открывает форму создания карточки
3. Продавец заполняет обязательные поля: название, категория, цена, количество
4. Продавец загружает фото (минимум 1, максимум 10)
5. Продавец заполняет описание (до 3000 символов)
6. Продавец нажимает «Сохранить как черновик» или «Отправить на модерацию»
7. Система сохраняет карточку и показывает подтверждение

**Альтернативный сценарий (невалидные данные):**

- На шаге 3–5 система проверяет заполнение. Если поля не заполнены или неверны — подсвечивает ошибки и не даёт отправить на модерацию. Сохранить черновик можно.

**Постусловие:** Карточка создана в статусе «Черновик» или «На модерации».

---

### UC-4: Провести модерацию

**Предусловие:** Товар отправлен на модерацию.

**Основной сценарий:**

1. Система автоматически проверяет: запрещённые слова, формат фото, корректность цены
2. Если авто-проверка пройдена — карточка поступает к модератору
3. Модератор проверяет соответствие правилам площадки
4. Модератор нажимает «Одобрить»
5. Система меняет статус на «Опубликован» и выводит товар на витрину

**Альтернативный сценарий (отклонение):**

- На шаге 4 модератор нажимает «Отклонить», выбирает причину из списка (или пишет свою)
- Система меняет статус на «Отклонён», отправляет уведомление продавцу

---

## 5. Примечания

- Статусы товара: Черновик → На модерации → Опубликован / Отклонён
- При отклонении товар возвращается к статусу, позволяющему редактирование
- Снятый с публикации товар сохраняет все данные и может быть повторно опубликован
