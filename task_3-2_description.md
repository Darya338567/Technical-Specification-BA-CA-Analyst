# Задание 3.2: Пошаговый алгоритм создания пользователя (бэкенд)

## Вход
POST /api/v1/auth/register с телом запроса (JSON)

---

## Шаг 1. Получить запрос
- Прочитать JSON из тела запроса
- Если тело пустое или не JSON → вернуть 400 Bad Request

---

## Шаг 2. Проверить reCAPTCHA
- Отправить `recaptchaToken` в сервис Google reCAPTCHA
- Если проверка не пройдена → вернуть 400 с ошибкой «Please verify reCaptcha to register!»
- Если сервис reCAPTCHA недоступен → вернуть 500

---

## Шаг 3. Проверить, что все обязательные поля заполнены
- Проверить наличие: firstName, lastName, username, password
- Если какое-то поле отсутствует или пустое → вернуть 400 с указанием, какого поля не хватает

---

## Шаг 4. Проверить формат каждого поля

| Поле | Что проверяем | Если не прошло |
|:---|:---|:---|
| firstName | Только буквы, 1–50 символов | 400 |
| lastName | Только буквы, 1–50 символов | 400 |
| username | 3–30 символов, буквы/цифры/`_` | 400 |
| password | Мин. 8 симв., 1 цифра, 1 заглавная, 1 строчная, 1 спецсимвол | 400 |

Если несколько полей невалидны → собрать все ошибки в `fieldErrors` и вернуть одним 400

---

## Шаг 5. Проверить, не занят ли username
- Сделать запрос в БД: `SELECT * FROM users WHERE username = ?`
- Если пользователь найден → вернуть 409 Conflict с текстом «User exists!»

---

## Шаг 6. Хешировать пароль
- Применить bcrypt (или аналог) к `password`
- Получить `passwordHash`

---

## Шаг 7. Создать пользователя в БД
- Вставить запись: firstName, lastName, username, passwordHash
- Получить сгенерированный `userId` (UUID или автоинкремент)

---

## Шаг 8. Вернуть успешный ответ
- HTTP 201 Created
- Тело:

    {
      "userId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "username": "ivan",
      "message": "User registered successfully"
    }

---

## Блок-схема алгоритма (текстовая)

    [Получить POST-запрос]
            ↓
    [Тело запроса валидно?] ──Нет──→ [400 Bad Request]
            ↓ Да
    [Проверить reCAPTCHA] ──Нет──→ [400 reCAPTCHA failed]
            ↓ Да
    [Все поля заполнены?] ──Нет──→ [400 + список недостающих полей]
            ↓ Да
    [Поля по формату?] ──Нет──→ [400 + fieldErrors]
            ↓ Да
    [Username свободен?] ──Нет──→ [409 User exists!]
            ↓ Да
    [Хешировать пароль (bcrypt)]
            ↓
    [Сохранить в БД]
            ↓
    [201 Created + userId, username, message]

---

## Примечания для разработчика

1. Порядок проверок важен: reCAPTCHA → обязательные поля → формат → уникальность username
2. Если username занят — не проверяем остальные поля, сразу 409
3. Пароль никогда не храним в открытом виде, только хеш
4. Все ошибки 400 возвращаем с понятным текстом для фронтенда
